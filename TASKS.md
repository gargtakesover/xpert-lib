# Xpert Feature Pipeline: Phase 3 (Semantic Extraction)

## Overview
This document catalogs the upcoming feature development pipeline for the `xpert` extraction library. These tasks leverage the newly discovered DOM components generated by recent upstream Nitter backend updates.

## Deep Semantic Node Tasks
- [ ] **Implement Community Notes Extraction:**
  - **Objective:** Detect crowdsourced "Fact Checks".
  - **Execution:** Target the `.community-note` DOM elements within Tweet bodies during `_parse_tweet`.
  - **Schema:** Map `community_note: str` and `has_community_note: bool` into the `Tweet` payload.

- [ ] **Image Accessibility (Alt Text) Parsing:**
  - **Objective:** Extract the original author's semantic description of visual content.
  - **Execution:** Update image iteration logic to capture the `alt="..."` HTML attribute alongside the source URI.
  - **Schema:** Refactor `images: List[str]` into `images: List[dict]` natively.

- [ ] **Algorithmic Disclosures (AI/Ads):**
  - **Objective:** Filter artificial or paid ecosystem clutter.
  - **Execution:** Detect the new Nitter disclosure wrappers.
  - **Schema:** Map `is_ai_generated: bool` and `is_promoted: bool`.

- [ ] **Tweet Edit History Tracking:**
  - **Objective:** Detect dynamically shifting chronological narratives.
  - **Execution:** Sniff the tweet chronological datestamps for the `Edited` tag suffix.
  - **Schema:** Construct `is_edited: bool`.

- [ ] **Grok AI Share Cards:**
  - **Objective:** Intercept structured Premium Grok conversations.
  - **Execution:** Target the Grok-specific URL boundary components inside the tweet payload.
  - **Schema:** Map `grok_share: str` directly into the extraction pipeline.
